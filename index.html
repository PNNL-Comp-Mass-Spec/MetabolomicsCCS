Filters and Sorting
Filters are a way to slim down the data in the table.
The pathway filter will filter the results of the table to show only metabolites loacated in that pathway. Similarly the class and subclass filter will filter the results of the table to show only metabolites loacated in that class or subclass. However, you can choose multiple classes or subclasses. With each of the same type added to the filter it will add the metabolites found with that type. That means it is an "or" operation, where you look for metabolites that have this subclass or that subclass. An example would be Amino Acid and Fatty Acid in the filter, it will show all metabolites that have amino acid or fatty acid as a subclass. The filtering between class and subclass is an "and" opertion, where you look for metabolites that have this class and that subclass. An example would be Secondary Metabolite as the class and Amino Acid as the subclass, it will only show metabolites that have both secondary metabolite as the class and amino acid as the subclass.
You can also type in a name, class, subclass, kegg id, cas number, formula, or exact mass into the search box.
There are several ways to sort the data in the table, default it sorts the data based on class and subclass. Clicking on the class will change the order the class is sorted but, not the subclass. Clicking the subclass will do the same sorting based on subclass, but not class. You can also sort by name, cas number, formula, exact mass, and CCS. When sorting any filters will affect the results of the ordering.
Downloading Data
Being able to download the data for this project is very important to us. We want everyone to use this data as a standard for identifying metabolites from the collision cross section. To download the data click the download button and choose the formate that is appropriate for you. Simple, right?
